# Skill: Plan & Review (תכנון וביקורת)

## מתי להשתמש
לפני כל משימה חדשה - פיצ'ר, באג, רפקטורינג.

## תהליך

### שלב 1: כתיבת תכנית
1. קרא את `.claude/plan-template.md`
2. מלא את כל הסעיפים
3. רשום את כל הקבצים שצריך לשנות
4. פרט את שלבי הביצוע

### שלב 2: ביקורת (Senior Review)
הפעל בדיקה עם הפרומפטים הבאים:

```
"בדוק את התכנית הזו כמהנדס בכיר. חפש:
1. מקרים קצוניים שלא טופלו
2. בעיות ביצועים פוטנציאליות
3. בעיות אבטחה
4. תלויות שלא זוהו
5. דרך פשוטה יותר להשיג את אותה תוצאה"
```

### שלב 3: עדכון התכנית
שלב את ההערות מהביקורת ועדכן את התכנית.

### שלב 4: יישום
רק אחרי שהתכנית מאושרת - התחל לקודד.

## דגשים
- לא מתחילים לקודד לפני שיש תכנית ברורה
- כל שינוי ב-API צריך לציין את הקבצים המושפעים
- תמיד לציין מה צריך לבדוק ידנית
