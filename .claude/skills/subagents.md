# Skill: Subagents (סוכנים משניים)

## מתי להשתמש
- משימות מורכבות שדורשות הרבה הקשר
- כשרוצים לשמור על חלון ההקשר של הסוכן הראשי נקי
- כשצריך לנתח מידע רב במקביל

## איך להפעיל
הוסף בתחילת הפרומפט:
```
use subagents
```

## דוגמאות

### רפקטורינג רחב
```
"use subagents

בצע רפקטורינג של מערכת הניהול של קריאות שירות.
פרק את המשימה לסוכנים:
1. סוכן לניתוח המצב הנוכחי
2. סוכן לתכנון הארכיטקטורה החדשה
3. סוכן ליישום השינויים
4. סוכן לביקורת קוד"
```

### ניתוח מקיף
```
"use subagents

נתח את כל הקומפוננטות ב-src/components/ ומצא:
- קוד כפול
- קומפוננטות לא בשימוש
- בעיות ביצועים
- חוסר עקביות בעיצוב"
```

### פיצ'ר גדול
```
"use subagents

הוסף מערכת התראות בזמן אמת.
סוכן 1: חקור את מערכת ההתראות הקיימת
סוכן 2: תכנן את הארכיטקטורה
סוכן 3: יישם את הבקאנד
סוכן 4: יישם את הפרונטאנד
סוכן 5: בדוק את הכל"
```

## כללים
- כל סוכן מקבל משימה ממוקדת וברורה
- הסוכן הראשי מנהל ומתאם
- תוצאות מסוכנים משניים מגיעות מסוכמות
- לא צריך להעביר הקשר מלא - מספיק תיאור ממוקד
