# Skill: Code Review (ביקורת קוד)

## מתי להשתמש
לפני merge של קוד חדש, או כשרוצים לבדוק קוד קיים.

## פרומפטים לביקורת

### ביקורת כללית
```
"בצע code review מקיף על השינויים שעשינו. בדוק:
1. קריאות הקוד
2. ביצועים
3. אבטחה (XSS, injection, חשיפת נתונים)
4. טיפול בשגיאות
5. תאימות RTL
6. נגישות (a11y)
7. שימוש נכון ב-React patterns"
```

### ביקורת אבטחה
```
"בדוק את הקוד הזה מבחינת אבטחה. חפש:
- חשיפת מידע רגיש
- XSS vulnerabilities
- SQL injection
- CSRF
- בעיות הרשאות
- חשיפת API keys"
```

### ביקורת ביצועים
```
"בדוק את הקוד מבחינת ביצועים:
- renders מיותרים
- missing memoization
- bundle size impact
- network calls מיותרים
- memory leaks"
```

### אתגר (Challenge Mode)
```
"אתגר אותי על השינויים האלה. הוכח לי שהם באמת עובדים כמו שצריך.
מצא את הבעיה הכי גדולה בקוד הזה."
```

## דגשים ספציפיים ל-NatID CRM
- וודא שימוש נכון ב-React Query (לא state מקומי לנתוני שרת)
- וודא שכל הטקסט בעברית
- וודא RTL תקין בכל רכיבי ה-UI
- וודא שימוש ב-path aliases (@/)
- וודא שימוש בקומפוננטות shadcn/ui קיימות
