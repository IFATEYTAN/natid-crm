# Skill: Prompt Patterns (דפוסי פרומפט)

## פרומפטים חזקים לשימוש יומיומי

### 1. Challenge Mode (מצב אתגר)
```
"אתגר אותי על השינויים האלה ותבדוק אותי"
```
```
"הוכח לי שזה באמת עובד כמו שצריך"
```
```
"מצא את הבאג הכי קריטי בקוד הזה"
```

### 2. Simplicity Mode (מצב פשטות)
```
"זרוק הכל ויישם את הפתרון הכי אלגנטי ופשוט שאתה יכול לחשוב עליו"
```
```
"פשט את הקוד הזה למקסימום בלי לפגוע בפונקציונליות"
```

### 3. Expert Mode (מצב מומחה)
```
"התנהג כמו senior React developer עם 15 שנות ניסיון.
בדוק את הקוד הזה ותן feedback כנה וישיר."
```
```
"כמהנדס אבטחה - האם הקוד הזה בטוח? מצא כל vulnerability."
```

### 4. Teaching Mode (מצב הוראה)
```
"הסבר לי למה בחרת בפתרון הזה ולא באלטרנטיבות"
```
```
"ספר לי מה הסיכונים של הגישה הזו"
```

### 5. Documentation Mode (מצב תיעוד)
```
"עדכן את מסמכי הפרויקט כדי שלא נחזור על הטעות הזו לעולם"
```
```
"כתוב סיכום של מה שלמדנו היום ושמור ב-LESSONS_LEARNED.md"
```

### 6. Analysis Mode (מצב ניתוח)
```
"נתח את הקודבייס ותן לי את 5 הבעיות הכי דחופות לטיפול"
```
```
"מה ישתבש אם נוסיף 1000 משתמשים במקביל?"
```

### 7. Planning Mode (מצב תכנון)
```
"תכנן את הפיצ'ר הזה מא' עד ת' לפני שנתחיל לקודד"
```
```
"מה כל הדברים שיכולים להשתבש עם השינוי הזה?"
```

## עקרונות
1. **ככל שהפרומפט יותר ספציפי - התוצאה יותר טובה**
2. **תמיד ציין את ההקשר** - איזה קובץ, איזה מודול, מה המטרה
3. **בקש הוכחות** - "הוכח", "הראה", "תן דוגמה"
4. **אתגר** - אל תקבל את התשובה הראשונה, בקש alternatives
5. **השתמש בהכתבה קולית** - fn+fn ב-macOS, מדברים הרבה יותר מהר
