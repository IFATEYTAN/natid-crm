# Skill: Update Documentation (עדכון מסמכים)

## מתי להשתמש
- אחרי תיקון באג משמעותי
- אחרי הוספת פיצ'ר חדש
- כשמגלים דפוס חוזר של טעויות
- כשלומדים משהו חדש על המערכת

## מה לעדכן

### 1. LESSONS_LEARNED.md
```bash
# פתח את הקובץ
docs/LESSONS_LEARNED.md
```
הוסף רשומה חדשה עם:
- **תאריך**
- **קטגוריה** (Bug/Feature/Architecture/Performance/Security)
- **תיאור הבעיה**
- **הפתרון**
- **הלקח** - מה לעשות/לא לעשות בעתיד

### 2. SYSTEM_SPECIFICATION.md
עדכן כשיש:
- שינוי במבנה הנתונים
- פיצ'ר חדש שמשנה את הארכיטקטורה
- אינטגרציה חדשה

### 3. CLAUDE.md
עדכן כשיש:
- שינוי בכלי פיתוח
- Convention חדש
- שינוי במבנה התיקיות

### 4. docs/WORKFLOWS.md
עדכן כשיש:
- תהליך עסקי חדש
- שינוי בתהליך קיים

## הפרומפט
```
"עדכן את מסמכי הפרויקט עם הלקח הבא: [תאר את הלקח].
ודא שזה לא יחזור על עצמו."
```

## דגשים
- **מסמכים איכותיים = פחות טעויות של Claude**
- כל עדכון מסמך מגדיל את האיכות של כל סשן Claude עתידי
- כתוב בעברית בריאה וברורה
