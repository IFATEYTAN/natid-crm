# פונקציות מתוזמנות (Scheduled Functions)

מספר פונקציות backend נועדו לרוץ **על לוח זמנים** (cron), ולא כתגובה לבקשת משתמש.
ב-repo אין הגדרת מתזמן — יש להגדיר אותן בלוח הבקרה של Base44 (Scheduler / Cron),
או להסתמך על ה-poller בצד-לקוח במקומות שבהם הוא קיים.

> חשוב: כל עוד המתזמן בפלטפורמה לא מוגדר, התהליכים הללו לא ירוצו אוטומטית בצד שרת.
> עבור פקיעת הצעות שיבוץ קיים גם **poller בצד-לקוח** (`src/hooks/useStaleAssignmentPoller.js`)
> שמורכב ב-`Layout` ורץ כל ~2 דקות כשטאב של אדמין/מוקדן פתוח — רשת ביטחון בלבד,
> לא תחליף למתזמן production.

## הפונקציות והתדירות המומלצת

| פונקציה | מה היא עושה | תדירות מומלצת | Auth |
|---------|-------------|----------------|------|
| `syncNatiData` | מסנכרן קריאות/תיקים/ספקים/לקוחות פתוחים מ-DB של נתי (עד 30 לריצה, בעדיפות לחדשים ואז מעודכנים) | כל 5–10 דקות | cron (עם `automation_key`) או admin |
| `processStaleAssignments` | פוקעת הצעות שיבוץ שלא אושרו, מציעה מחדש לספק הבא, ומסלימה לתור התפעול אחרי 30 דק' | כל 1–2 דקות | cron ללא משתמש, או admin/operator |
| `nudgeStaleVendorLocations` | התראה לספקים שמיקום ה-GPS שלהם לא עודכן | כל 5–10 דקות | cron / admin |
| `detectSmartAlerts` | זיהוי חריגות/עיכובים ויצירת התראות | כל 2–5 דקות | cron / admin |
| `checkContractExpiry` | התראה על חוזי ספקים שעומדים לפוג | יומי | cron / admin |
| `checkDepositExpiry` | התראה על פיקדונות שעומדים לפוג | יומי | cron / admin |
| `autoTransferAgentCalls` | העברת קריאות מנציג שיצא להפסקה/לא זמין | כל 2–5 דקות | cron / admin |
| `closeStaleNatiCalls` | סגירה אוטומטית של קריאות נתי ישנות | יומי | cron / admin |

> `syncNatiData` תומך גם ב-`automation_key` (סוד `SYNC_AUTOMATION_KEY`) כפרמטר גוף הבקשה,
> לשימוש כשקשה להגדיר הרשאת admin/operator ל-cron. אפשר להשאיר את גוף הבקשה `{}` אם
> ה-Scheduler שולח בקשה מאומתת בתור admin, אחרת לשלוח `{ "automation_key": "..." }`.

## כיצד להגדיר ב-Base44

1. היכנס ללוח הבקרה של האפליקציה ב-Base44 → אזור ה-Functions / Scheduler.
2. לכל פונקציה מהטבלה, צור Schedule (cron) בתדירות המומלצת.
3. ודא שה-Schedule מריץ את ה-endpoint ללא גוף בקשה מיוחד (`{}`), אלא אם צוין אחרת.
4. לאחר ההגדרה — אפשר לכבות/להסיר את ה-poller בצד-לקוח אם רוצים (לא חובה; הוא מוגן מפני
   ריצה כפולה ומדלג כשהוא כבר רץ).

## בדיקה ידנית

ניתן להריץ כל פונקציה ידנית (כ-admin/operator) דרך:

```js
await base44.functions.invoke('processStaleAssignments', {});
```
