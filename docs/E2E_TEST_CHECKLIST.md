# בדיקת End-to-End - מסע מלא של קריאת שירות

> **מטרה**: לוודא שמסע הקריאה מקצה לקצה עובד - מפתיחת הקריאה במוקד עד למשוב הלקוח, כולל כל כלי ה-AI, GPS, תמונות, והודעות.

**זמן משוער לכל הריצה**: 30-45 דקות
**משתתפים מומלצים**: 2 אנשים (אחד על Operator, אחד על Vendor) או אחד עם 2 מסכים

---

## שלב 0: הכנות (פעם אחת)

### חשבונות בדיקה
- [ ] משתמש Operator קיים, סיסמה ידועה
- [ ] משתמש Vendor קיים, סיסמה ידועה, **משויך לרשומת ספק** (Vendor entity קיים)
- [ ] אם רוצים לבדוק SMS ללקוח: לקוח אמיתי (עם טלפון אמיתי) שיוכל לקבל הודעות

### תשתיות מוגדרות
- [ ] Base44 Secrets מוגדרים:
  - [ ] `NATI_API_JWT_TOKEN`
  - [ ] `NATI_API_CLIENT_ID`
  - [ ] `TWILIO_ACCOUNT_SID` (לבדיקת SMS)
  - [ ] `TWILIO_AUTH_TOKEN`
  - [ ] `TWILIO_FROM_NUMBER`
- [ ] Push Notifications מאופשרות בדפדפן

### מכשירים
- [ ] **מחשב** - לדפדפן Operator
- [ ] **טלפון נייד אמיתי** - לדפדפן Vendor (חובה ל-GPS ותמונות)
- [ ] **טלפון של "לקוח"** - לקבלת SMS ולמילוי משוב

## 🔧 פתרון תקלות נפוצות

| תופעה | סיבה אפשרית | פתרון |
|-------|--------------|--------|
| GPS לא עובד | דפדפן חוסם הרשאות | להגדיר ב-browser settings → location → allow |
| תמונות לא נשמרות | בעיית permissions ב-storage | לבדוק `Base44 storage` הגדרות |
| SMS לא נשלח | Twilio לא מוגדר | להגדיר Secrets + לוודא credit |
| Push notifications לא מגיעות | VAPID key חסר | להריץ `getVapidPublicKey` function |
| AI לא עונה | OpenAI / Base44 LLM לא מוגדר | לבדוק `Core.InvokeLLM` עובד |
| הקריאה לא עוברת לספק | autoAssignVendor שגוי | לבדוק לוגים של פונקציה |
