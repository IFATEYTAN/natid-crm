## 2. מה בדיוק תוקן בסעיף 10 — הזרימה החדשה

בדיוק לפי הניסוח של דורית מ-13.7:

| כשהקריאה במצב... | בבורר הסטטוס יופיעו רק |
|---|---|
| ספק שובץ / ספק בדרך | **ספק הגיע ללקוח** + ביטול פניה |
| ספק הגיע ללקוח / בטיפול | **ספק הגיע לאחסנה** + **סגירת קריאה** + ביטול פניה |
| באחסנה | **סגירת קריאה** (שם בוחרים: אחסנה נוספת / יעד סופי) + ביטול פניה |

הערות:
- "ספק הגיע ליעד" נרשם דרך **סגירת קריאה** ← "גרר הגיע ליעד הסופי בהצלחה" (אותו מסך סגירה שדורית אישרה בסעיף 9).
- "ביטול פניה" בבורר פותח את מסך הביטול המלא (סיבה + כללי עירבון).
- ההתנהגות מקובעת ב-24 בדיקות אוטומטיות שרצות על כל שינוי עתידי.

## 3. מה לבקש מדורית לבדוק בסבב הבא ✔️

1. **סעיף 10 (העיקרי):** לפתוח קריאה משובצת ← ללחוץ על בורר הסטטוס ← לוודא שמופיעות רק האפשרויות מהטבלה למעלה, בכל שלב בזרימה: שיבוץ ← הגיע ללקוח ← אחסנה/סגירה.
2. **ביטול בכל שלב:** לוודא ש"ביטול פניה" מופיע בכל שלב ופותח את מסך הביטול (עם סיבה).
3. **הקריאות החדשות:** הלוח מציג עכשיו ~155 קריאות אמיתיות מנתי — לוודא שהנתונים מוכרים לה (מספרי קריאה, ספקים, לקוחות). קריאות עם "לא צוין" כשם לקוח = פניות שאין להן שם מבקש בנתי (תקין).
4. **סעיף 8:** לפתוח קריאה משובצת ← "שבץ ספק" ← לראות את "המשך עם ספק נוסף" בתוך הדיאלוג — ולהגיד אם המיקום הזה מספיק או שצריך כפתור נפרד.
5. **קריאה חדשה ידנית:** לפתוח קריאה, לשבץ ספק, לוודא שה-SMS מגיע לספק ושהקריאה נכנסת לתור העבודה.
