# סקירה מקיפה של מערכת CRM נתיד

מסמך זה מספק סקירה מלאה ומפורטת של מערכת נתיד CRM - פלטפורמה מקיפה לניהול שירותי סיוע בדרכים ושירותי שטח, הבנויה על פלטפורמת Base44.

---

## 1. מבנה הפרויקט וארכיטקטורה

### מחסנית טכנולוגית

- **Frontend**: React 18.2 עם Vite 6.1
- **עיצוב**: Tailwind CSS 3.4.17 עם מערכת עיצוב מותאמת אישית
- **Backend**: Base44 SDK (פונקציות serverless מבוססות Deno)
- **ניהול State**: TanStack React Query 5.84.1
- **טיפול בטפסים**: React Hook Form 7.54.2 + וולידציה עם Zod
- **רכיבי UI**: Radix UI (ספרייה נרחבת של קומפוננטות)
- **גרפים**: Recharts 2.15.4 + Chart.js
- **מפות**: React Leaflet 4.2.1 (עם אינטגרציה ל-Waze)
- **תוכן עשיר**: React Quill, React Markdown, HTML2Canvas/jsPDF
- **אנימציות**: Framer Motion, Lottie React
- **התראות**: Sonner, React Hot Toast

### מבנה התיקיות

```
/home/user/natid-crm/
├── src/
│   ├── pages/          # 25 דפי קומפוננטות ראשיים
│   ├── components/     # קומפוננטות UI ודומיין לשימוש חוזר
│   ├── api/            # הגדרת Base44 SDK Client

│   ├── lib/            # כלי עזר, אימות, Query Client
│   └── utils/          # פונקציות עזר
├── functions/          # 9 פונקציות serverless בצד השרת
├── vite.config.js      # קונפיגורציית Vite עם פלאגין Base44
├── tailwind.config.js  # התאמה אישית של Tailwind
└── package.json        # תלויות וסקריפטים
```

## 2. ישויות מסד נתונים ומודלי נתונים

המערכת מנהלת 14 ישויות ליבה:

### ישויות ראשיות

#### Call - רשומות קריאות שירות/אירועים
**שדות:**
- `call_number` - מספר קריאה ייחודי
- פרטי לקוח
- מידע על רכב
- סוג תקלה/בעיה
- מיקום
- סטטוס
- הקצאת ספק שירות
- מידע זמנים
- דירוגים
- עלויות

#### Case - רשומות שירות חלופיות/משלימות
**שדות:**
- `case_number` - מספר תיק ייחודי
- שם לקוח
- סוג שירות
- מיקום
- סטטוס
- עדיפות
- ספק שירות מוקצה

#### Vendor (ספקי שירות)
**שדות:**
- `vendor_name` - שם הספק
- פרטי קשר
- סוגי שירותים
- אזורי כיסוי
- סטטוס זמינות
- דירוג
- תעריף תשלום
- שעות עבודה
- חוזים

#### Customer (לקוחות)
**שדות:**
- שם
- סוג (פרטי/ארגוני/חברת ביטוח)
- איש קשר
- טלפון
- אימייל
- כתובת
- סוג חוזה
- הגדרות SLA
- תקציב

#### User - משתמשי המערכת עם תפקידים (admin/user/vendor)
**שדות:**
- `full_name` - שם מלא
- אימייל
- טלפון
- תפקיד
- תאריך יצירה

### ישויות תומכות

- **VendorLocation** - מעקב GPS בזמן אמת אחר ספקי שירות
- **VendorRating** - דירוגים ומשוב של לקוחות על ספקים
- **VendorPayment** - רשומות תשלום וחשבוניות
- **VendorContract** - הסכמי שירות בין החברה לספקים
- **WorkQueue** - תור הקצאת קריאות למפעילים
- **CallHistory** - יומן ביקורת של כל השינויים בקריאות
- **CallPhoto** - תמונות המשויכות לקריאות
- **CaseActivity** - יומן פעילות לתיקים
- **ServiceProvider** - רשומות ספקי שירות חלופיים

## 3. תכונות ועמודים (25 סה"כ)

### דפי דשבורד ראשיים

#### Dashboard - סקירת KPI ראשית
- מדדי קריאות בזמן אמת (פתוחות, ממתינות, בטיפול, הושלמו)
- ניטור תור עבודה לפי סוכן
- התפלגות סטטוס קריאות (גרף עמודות)
- מגמת קריאות ל-7 ימים (גרף קווים)
- טבלת קריאות אחרונות
- מעקב SLA ומדדי שביעות רצון לקוחות

#### Calls - ניהול קריאות מקיף
- סינון מתקדם (סטטוס, טווח תאריכים, עיר, ספק, VIP)
- חיפוש לפי מספר קריאה, שם לקוח, טלפון, מספר רכב
- אינדיקטורים לסטטוס SLA
- מעקב זמן המתנה
- הקצאת ספקים
- עימוד (20 פריטים לעמוד)
- פונקציונליות ייבוא/ייצוא

#### Cases - ניהול תיקי שירות
- יצירה/עריכה/צפייה בתיקים
- מעקב סטטוס (חדש, הוקצה, בדרך, במקום, בטיפול, הושלם, בוטל)
- פילטרים לפי סוג שירות
- רמות עדיפות
- חיפוש מבוסס מיקום

### ניהול לקוחות

#### Customers - מסד נתוני לקוחות
- יצירה/עריכה/מחיקת לקוחות
- סוגי לקוחות: פרטי, ארגוני, חברת ביטוח, מוסך
- ניהול חוזים (חודשי, שנתי, לפי מקרה)
- הגדרות SLA (זמני תגובה והגעה)
- מעקב סטטוס (פעיל, לא פעיל, מושעה)
- מעקב תקציב חודשי

#### UserManagement - ניהול צוות
- מערכת הזמנות משתמשים
- ניהול תפקידים (admin, user, vendor)
- רשימת משתמשים וחיפוש
- הזמנות מבוססות אימייל

### ניהול ספקים/ספקי שירות

#### ServiceProviders - מדריך ספקי שירות
- יצירה/עריכה/מחיקת ספקים
- ניהול סוגי שירותים
- הגדרת אזורי כיסוי
- סטטוס זמינות (זמין, עסוק, לא מקוון, הפסקה)
- דירוגי ביצועים וסטטיסטיקות השלמה
- תעריפי תשלום לכל קריאה
- שעות עבודה ותמיכה 24/7
- צפייה בפרופילי ספקים

#### VendorProfile - פרטי ספק בודד
- סטטיסטיקות ומדדי ביצוע של הספק
- היסטוריית קריאות
- סיכום דירוג ומשוב
- יכולות שירות
- מידע ליצירת קשר

#### VendorPortal - פורטל שירות עצמי לספקים
- דשבורד לקריאות מוקצות
- תצוגת קריאות פעילות
- מדדי ביצועים שבועיים
- מעקב זמן תגובה
- ניווט למיקומי עבודה (אינטגרציה עם Waze)
- מערכת דירוג
- היסטוריית תשלומים

#### VendorPayments - מעקב תשלומים לספקים
- סיכומי תשלום חודשיים
- חיוב קריאות שהושלמו
- מעקב סטטוס תשלום (ממתין/שולם)
- גרפי היסטוריית תשלומים (מגמה ל-6 חודשים)
- חישובי עלויות

#### AllVendorsMap - מעקב מיקום ספקים בזמן אמת

## 4. נקודות קצה API ופונקציות Backend

למערכת יש 9 פונקציות Deno serverless שנפרסו על Base44:

### פונקציה 1: autoAssignVendor.ts
**מטרה:** הקצאה אוטומטית חכמה של ספקים לקריאות

**טריגר:** יצירת קריאה או הפעלה ידנית

**אלגוריתם:** מערכת ניקוד מתקדמת עם גורמים ממושקללים:
- **ניקוד מרחק (35%):** נוסחת Haversine, קנס על נתוני מיקום ישנים
- **ניקוד זמינות (25%):** סטטוס ספק (זמין/עסוק/לא מקוון)
- **ניקוד דירוג (20%):** דירוג ממוצע של ספק מלקוחות
- **ניקוד זמן תגובה (10%):** זמני תגובה היסטוריים
- **ניקוד חוזה (10%):** בונוס לחוזים פעילים

**תכונות:**
- התאמת סוג שירות לאזור כיסוי
- לוגיקת גיבוי אם אין התאמות מדויקות
- חישוב מרחק ו-ETA
- התראת SMS לספק המוקצה
- רישום היסטוריית קריאות עם פרטי ניקוד

### פונקציה 2: checkAndSendNotifications.ts
**מטרה:** מנוע התראות אוטומטי

**טריגר:** מתוזמן או הפעלה ידנית

**סוגי אירועים המנוטרים:**
- `call_unassigned` - קריאות ממתינות מעבר לסף
- `sla_near_breach` - קריאות המתקרבות למועד SLA
- `low_rating` - דירוגי לקוחות מתחת לסף
- `call_cancelled` - קריאות שבוטלו
- `vendor_delayed` - עיכובי הגעת ספק

**ערוצי התראה:**
- הודעות SMS
- התראות אימייל
- התראות בתוך האפליקציה (נרשמות בהיסטוריית הקריאה)

**אפשרויות סינון:**
- מבוסס עדיפות (דחוף/רגיל)
- אזור גיאוגרפי
- רדיוס מיקום (מבוסס GPS)

### פונקציה 3: 99digitalBot.ts
**מטרה:** אינטגרציה עם צ'אטבוט 99 Digital

**טריגר:** Webhook מפלטפורמת הבוט

**עיבוד נתונים:**
- חילוץ מידע על לקוח
- פרסור פרטי רכב
- מיפוי סוג אירוע
- חישוב אזור מעיר
- ניקוד עדיפות (בסיס 50, +30 VIP, +25 דחוף, +20 תאונה)

**יוצר:** ישות Call מלאה עם כל הפרטים משיחת הבוט

**תכונות:**
- וולידציה של שדות נדרשים
- יצירת קוד תגובה ללקוח
- מעקב ביטוח/מנוי
- אכלוס נתוני קריאה מקיף

### פונקציה 4: botWebhook.ts
**מטרה:** נקודת קצה webhook גנרית לבוט

**טריגר:** Webhook נכנס מכל מערכת בוט

**תכונות:**
- מיפוי שדות גמיש
- וולידציית נתונים
- יצירת קריאה מPayload של webhook
- טיפול בשדות אופציונליים חסרים בצורה חלקה

### פונקציה 5: calculateDistanceAndETA.ts
**מטרה:** תכנון מסלול וחישוב ETA

**מחשב:**
- מרחק דרך (לעומת מרחק קו אווירי)
- זמן הגעה משוער
- אופטימיזציית מסלול

**משתמש ב:** אינטגרציה עם שירות מיפוי (ככל הנראה Google Maps)

### פונקציה 6: externalCrmWebhook.ts
**מטרה:** שער אינטגרציה ל-CRM חיצוני

**טריגר:** Webhook מ-Salesforce, HubSpot או CRM מותאם אישית

**תכונות:**
- וולידציית סוד webhook
- מיפוי שדות מ-CRM חיצוני
- המרת סוג בעיה
- מיפוי עדיפות
- יוצר ישות Call מנתונים חיצוניים
- הקצאה אוטומטית לתור (אופציונלי)

### פונקציה 7: sendSMS.ts
**מטרה:** שירות שליחת SMS

**משמש על ידי:** הקצאה אוטומטית, התראות, התראות ספקים

**תכונות:**
- שליחת טקסט SMS
- וולידציה של מספר טלפון
- קידוד תווים לטקסט עברי

### פונקציה 8: submitVendorRating.ts
**מטרה:** מטפל בהגשת דירוג לקוחות

**שדות קלט:**
- דירוג כולל (1-5)
- דירוגים מפורטים: זמן תגובה, איכות שירות, מקצועיות, תקשורת
- טקסט משוב
- דגל השלמה בזמן
- דגל המלצה
- מקור דירוג (לקוח/מערכת)

**פעולות:**
- יוצר רשומת VendorRating
- מעדכן דירוג ממוצע של ספק
