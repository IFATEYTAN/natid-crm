## 1. סקירה כללית

הסבב הזה מתקן 4 באגים קריטיים (P0) ו-4 באגים בעדיפות גבוהה במנגנון הסנכרון מבסיס הנתונים של נתי שירותים אל ה-CRM. המסמך הזה מפרט כיצד לבדוק שכל תיקון אכן עובד.

### 1.1 רשימת התיקונים שיש לאמת

| # | קוד | תיאור | חומרה | קובץ עיקרי |
|---|-----|--------|--------|------------|
| F-01 | P0-MYSQL-LEAK | חיבורי MySQL נסגרים תמיד (try/finally) | 🔴 קריטי | `functions/syncNatiData.ts` |
| F-02 | P0-COOLDOWN | סנכרון נוסף בתוך 60 שניות נחסם בצד השרת | 🔴 קריטי | `functions/syncNatiData.ts` |
| F-03 | P0-EMPTY-GUARD | סגירת קריאות מנתי נעצרת אם ה-API מחזיר תוצאה ריקה חשודה | 🔴 קריטי | `functions/closeStaleNatiCalls.ts` |
| F-04 | P0-VENDOR-DEDUP | ספקים זוהים לפי `vendor_number` ולא לפי שם — מונע כפילויות | 🔴 קריטי | `functions/syncNatiData.ts` |
| F-05 | HI-TIMEZONE | תאריכים מנתי נקראים כ-Asia/Jerusalem ולא UTC | 🟠 גבוה | `functions/syncNatiData.ts` |
| F-06 | HI-POLL-RATE | סקרת CallChat ירדה מ-5s ל-30s (חיסכון בקריאות) | 🟠 גבוה | `src/features/calls/hooks/useCallChat.js` |
| F-07 | HI-A11Y | aria-labels וטבעות פוקוס ב-NatiAssistant | 🟠 גבוה | `src/components/calls/NatiAssistant.jsx` |
| F-08 | HI-RTL-TOAST | toaster של Sonner מוצג ב-RTL | 🟠 גבוה | `src/main.jsx` או `src/Layout.jsx` |

### 1.2 מה לא נבדק במסמך הזה

- שינויים שאינם קשורים לסנכרון Nati (Calls UI, Vendors UI כלליים)
- ביצועים תחת עומס (לא בסקופ עכשיו)
- תאימות דפדפנים (נבדק בנפרד)

## 3. תרחישי בדיקה

### TC-01 — כניסה למסך הסנכרון

**מטרה:** לוודא שאדמין רואה את כרטיס הסנכרון ומשתמשים אחרים לא.

| צעד | פעולה | תוצאה מצופה |
|-----|--------|--------------|
| 1 | התחבר כ-Admin | מגיע ל-Dashboard |
| 2 | נווט ל-**הגדרות → אינטגרציות** | רואה כרטיס "סנכרון מנתי שירותים" עם תג צהוב "דמו / Read-only" |
| 3 | התנתק והתחבר כ-Operator | נווט לאותו URL — **לא רואה** את הכרטיס |

✅ **קריטריון קבלה:** הכרטיס מוצג רק ל-Admin. ל-Operator/Agent/Vendor המסך מציג גישה מוגבלת.

---

### TC-02 — Dry Run (תצוגה מקדימה)

**מטרה:** לוודא שמשיכת נתונים לקריאה בלבד עובדת.

| צעד | פעולה | תוצאה מצופה |
|-----|--------|--------------|
| 1 | במסך אינטגרציות, לחץ **"תצוגה מקדימה (Dry Run)"** | spinner על הכפתור |
| 2 | המתן עד 30 שניות | בלוק כחול מופיע: "תוצאות תצוגה מקדימה" |
| 3 | ודא שכל 4 הספירות מוצגות | סה״כ ב-API / יסונכרנו / ספקים / לקוחות — כולם מספרים גדולים מ-0 |
| 4 | פתח את "דוגמה לקריאה אחת שתסונכרן" | רואה JSON עם פרטי קריאה (לא ריק) |

✅ **קריטריון קבלה:** התוצאות מוצגות, כל המספרים סבירים, **לא** נוצרו רשומות חדשות במערכת.

---

### TC-03 — סנכרון אמיתי (Real Sync)

**מטרה:** לוודא שסנכרון מכניס נתונים תקינים.

| צעד | פעולה | תוצאה מצופה |
|-----|--------|--------------|
| 1 | רשום את מספר הקריאות הנוכחי במערכת | מספר X |
| 2 | לחץ **"בצע סנכרון"** | spinner על הכפתור |
| 3 | המתן עד 60 שניות | toast הצלחה ירוק: "הסנכרון הושלם — N רשומות..." |
| 4 | נווט ל-**קריאות** | מספר הקריאות גדל ב-N (X+N) |
| 5 | פתח קריאה חדשה | יש פרטים: לקוח, ספק, כתובת, תאריך |
| 6 | ודא תאריך הקריאה | התאריך **בשעון ישראל** (לא UTC, לא 3 שעות פחות) ← **F-05** |

✅ **קריטריון קבלה:** רשומות חדשות נכנסו, הפרטים מלאים, התאריך תקין.

---

### TC-04 — Cooldown (חסימת סנכרון כפול) ← **F-02 הקריטי**

**מטרה:** לוודא שסנכרון נוסף בתוך 60 שניות נחסם.

| צעד | פעולה | תוצאה מצופה |
|-----|--------|--------------|
| 1 | מיד אחרי שסיים TC-03, לחץ שוב **"בצע סנכרון"** | spinner קצר |
| 2 | תוך 5 שניות | **toast שגיאה אדום:** "יש להמתין... שניות לפני סנכרון נוסף" |
| 3 | המתן 60 שניות מסיום הסנכרון הקודם | |
| 4 | לחץ שוב **"בצע סנכרון"** | רץ בהצלחה (cooldown פג) |

✅ **קריטריון קבלה:** הסנכרון השני נחסם, ההודעה ברורה למשתמש, אחרי 60 שניות מותר שוב.

⚠️ **באג אם:** הסנכרון השני רץ בהצלחה (זה ה-bug המקורי) — דווח מיד.

---

### TC-05 — Vendor Dedup ← **F-04 הקריטי**

**מטרה:** לוודא שספקים זהים לא נכפלים.

**דורש:** בנתי שירותים יש לפחות 2 ספקים שונים עם אותו שם (`vendor_name`) אבל `vendor_number` שונה.

| צעד | פעולה | תוצאה מצופה |
|-----|--------|--------------|
| 1 | לפני הסנכרון, לך ל-**ספקים** | רשום את כמות הספקים |
| 2 | בצע סנכרון מלא | |
| 3 | חזור ל-**ספקים**, חפש את הספקים בעלי השם החוזר | רואה **שני ספקים** (לא אחד), אך כל אחד עם פרטי קשר שונים |
| 4 | פתח כל ספק | ה-`vendor_number` שונה בין השניים |

✅ **קריטריון קבלה:** ספקים אינם נדרסים זה על זה. כל `vendor_number` שונה = רשומה נפרדת.

⚠️ **באג אם:** רואים רק ספק אחד עם נתוני האחרון שסונכרן — זה הבאג המקורי.

---

### TC-06 — Dashboard מתעדכן ללא F5 ← **תיקון React Query**

**מטרה:** לוודא שלאחר סנכרון, ה-Dashboard מציג את הנתונים החדשים אוטומטית.

| צעד | פעולה | תוצאה מצופה |
|-----|--------|--------------|
| 1 | פתח את ה-Dashboard ב-tab אחד | רואה את כל המספרים והגרפים |
| 2 | רשום את "סה״כ קריאות פתוחות" | מספר Y |
| 3 | חכה לסיום cooldown (60 שניות מסנכרון אחרון) | |
| 4 | לחץ על כפתור "סנכרן מנתי" ב-Dashboard עצמו | spinner |
| 5 | המתן לסיום (toast) | |
| 6 | בלי לרענן את הדף (אסור F5!) | המספרים מתעדכנים אוטומטית בתוך 2-3 שניות |

✅ **קריטריון קבלה:** הנתונים מתעדכנים בלי refresh ידני.

⚠️ **באג אם:** המספרים נשארים זהים עד שמרעננים את הדף — בעיית React Query invalidation.

---

### TC-07 — Closing stale calls ← **F-03**

**מטרה:** לוודא שסגירת קריאות ישנות לא חוסלת רשומות בטעות.

**דורש:** גישה ל-cron jobs ב-Base44 (אדמין).

| צעד | פעולה | תוצאה מצופה |
|-----|--------|--------------|
| 1 | בקונסולת Base44, רץ ידנית את `closeStaleNatiCalls` | מקבל תוצאה JSON |
| 2 | בדוק את שדה `aborted` | אם API נתי מחזיר ריק חשוד — `aborted: true` |
| 3 | בדוק שאף קריאה לא נסגרה בטעות | סופר את הקריאות הפתוחות לפני ואחרי — אותו מספר |

✅ **קריטריון קבלה:** הפונקציה לא סוגרת קריאות אם ה-snapshot מנתי לא מהימן.

---

### TC-08 — Polling rate of CallChat ← **F-06**

**מטרה:** לוודא שצ'ט הקריאות לא מציף את ה-API.

| צעד | פעולה | תוצאה מצופה |
|-----|--------|--------------|
| 1 | פתח DevTools → Network → סנן ל-`call-chat` או דומה | |
| 2 | פתח קריאה כלשהי, לחץ על הצ'ט | |
| 3 | המתן 2 דקות עם הדף פתוח | רואה ~4 קריאות (אחת כל 30 שניות), **לא** 24 (כל 5 שניות) |

✅ **קריטריון קבלה:** קריאות הסקרה מתבצעות אחת ל-30 שניות.

---

### TC-09 — נגישות וכיווניות ← **F-07, F-08**

**מטרה:** לוודא שתיקוני RTL ו-a11y עובדים.

| צעד | פעולה | תוצאה מצופה |
|-----|--------|--------------|
| 1 | פתח את העוזר החכם (NatiAssistant) — לחץ על האייקון הצף | רואה את העוזר |
| 2 | לחץ Tab על המקלדת לעבור בין כפתורים | טבעת פוקוס כחולה ברורה |
| 3 | פתח DevTools → Accessibility | לכפתורי האייקונים יש `aria-label` בעברית |
| 4 | גרום ל-toast להופיע (לדוגמה: בצע סנכרון) | ה-toast מופיע **מצד שמאל** (לא ימין) ועם טקסט מיושר ל-RTL |

✅ **קריטריון קבלה:** Tab עובר על כל הכפתורים, יש a11y labels, ה-toaster ב-RTL.
