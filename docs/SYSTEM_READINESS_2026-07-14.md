## פירוט לפי רכיב

### 1. קליטת קריאה חדשה — בוט / CRM‏ 🟢
**מה בנוי:** שלושה ערוצי קליטה: (א) **סנכרון ישיר ממסד הנתונים של נתי** — הערוץ הפעיל, מושך פניות פתוחות/סגורות, יוצר קריאות+תיקים ומקשר לקוחות וספקים אוטומטית; (ב) webhook לבוט 99digital — בנוי ומוגן, כרגע רדום-נעול (הוחלף בסנכרון הישיר); (ג) webhook ל-CRM חיצוני — בנוי, רדום-נעול; (ד) יצירה ידנית במוקד.
**בדיקות:** סנכרון מלא אומת חי ב-14.7 — 155/155 קריאות פתוחות מנתי, כולל תיקון באג (שם לקוח חסר) שנמצא ותוקן באותו יום. ‏5 בדיקות E2E אוטומטיות על פאנל הסנכרון.
**פער:** אין — הערוץ הפעיל עובד ומאומת.

### 2. שיוך אוטומטי למוקדן / תורי עבודה 🟡
**מה בנוי:** קריאה חדשה נכנסת אוטומטית ל-WorkQueue עם ניקוד עדיפות (דחיפות + VIP), משויכת למוקדן במשמרת עם העומס הנמוך ביותר, ונשלחות התראות. ניהול משמרות קיים.
**בדיקות:** יצירת קריאה + סנכרון Call→WorkQueue אומתו (9.7). תורי העבודה נוקו ותוקנו היום (189 רשומות).
**פער:** בדיקת עשן לשיוך המוקדן האוטומטי על קריאה אמיתית — מתוכננת ליום העבודה הראשון.

### 3. שיוך ספק בהתאם להגדרות 🟢
**מה בנוי:** דיאלוג שיבוץ אחוד: בחירת סוג רכב שירות (ניידת/גרר/גרר+אחסנה) עם סינון ספקים, שיבוץ אוטומטי לפי מיקום (מנוע ניקוד צד-שרת), המלצת AI, יעדי אחסנה ויעד סופי, החלפת ספק או קריאת המשך לספק נוסף, ו-SMS אוטומטי לספק בשיבוץ (מוגן מכפילויות).
**בדיקות:** דורית אישרה (סעיף 7, 13.7); תוקן ואומת סינון הספקים (9.7); סיווגי שירות מולאו ל-23 ספקים אמיתיים.
**פער:** אין — סעיפים 1 ו-8 של דורית נסגרו ב-15.7 (מצאה את "המשך עם ספק נוסף" והבינה את התנהגות הסינון). הקשחת סינון עתידית אפשרית אחרי סיווג מלא של הספקים.

### 4. ניהול הקריאה צד ספק + צד לקוח 🟡
**מה בנוי:** מסך ניהול קריאה מלא: זרימת סטטוסים מדורגת (החדש מ-14.7 — רק הצעד הבא האפשרי), סגירה עם 7 תוצאות טיפול, קריאות המשך אוטומטיות, ביטול עם כללי עירבון, צ'אט, תמונות, חתימה, היסטוריה מלאה. צד לקוח: פורטל מעקב (טלפון+מס' קריאה), SMS בשינויי סטטוס, סקר שביעות רצון.
**בדיקות:** דורית אישרה שיבוץ וסגירה (7, 9) והזרימה המדורגת פורסה (10); 24 בדיקות יחידה מקבעות את הזרימה; מחזור חיים מלא קיים כבדיקת E2E.
**פערים:** נוסחי ה-SMS ללקוח עדיין זמניים (ממתינים לנוסח סופי מהלקוח); בדיקת צד-לקוח אמיתית (SMS מגיע לטלפון) — ביום עבודה אמיתי.

### 5. מפה, זמינות ומיקומים אונליין 🟠
**מה בנוי:** מפת כל הספקים, מפת מעקב חי לקריאה (Leaflet), שיתוף מיקום GPS מצד הספק, חישובי מרחק ו-ETA.
**בדיקות:** 5 בדיקות E2E על מסכי ה-GPS (מבניות עוברות); הרינדור נבדק.
**פער:** ‏GPS חי מנייד אמיתי של ספק — 11 בדיקות מהמסמך של הצוות דורשות מכשיר בשטח. זה הרכיב הכי פחות מאומת.

### 6. ניהול התראות ואיחורים 🟢
**מה בנוי:** פאנל "ניהול איחורים" בדשבורד (חישוב SLA, סימון קריטי), התראות מערכת (SLA, אי-שיוך), פעמון התראות, התראות Push (תשתית), ג'ובים מתוזמנים לפקיעת חוזים ועירבונות (הוקשחו היום).
**בדיקות:** הפאנל הוכיח את עצמו בשטח ב-14.7 — זיהה 59 איחורים אמיתיים בקריאות שסונכרנו מנתי; פעמון ההתראות נבדק (9.7).
**פער:** ‏Push למכשיר אמיתי — דורש טלפון.

### 7. דוחות 🟡
**מה בנוי:** מסך דוחות, דוח ביצועי ספקים, דוח ניצול קריאות, ייבוא נתונים היסטוריים, גרפים בדשבורד.
**בדיקות:** טעינת הדוחות נבדקה (9.7, חלקי); ניזונים כעת מ-383 קריאות סגורות אמיתיות.
**פער:** אימות תוכן הדוחות מול נתוני אמת — מומלץ אחרי שבוע עבודה אמיתי, כשיהיו נתונים משמעותיים.

### 8. פורטל ספקים 🟡
**מה בנוי:** תפקיד ספק עם תיחום נתונים (רואה רק את הקריאות שלו), קבלת/דחיית הצעת שיבוץ, עדכון סטטוסים, פרופיל, זמינות והפסקות, מיקום.
**בדיקות:** באג התיחום הקריטי תוקן ואומת חי (9.7); התחברות ספק ותיחום נבדקים אוטומטית ב-E2E.
**פער:** זרימת ספק מלאה (קבלת קריאה בנייד ← טיפול ← סיום) עם ספק אמיתי — בשבוע הראשון.

## התהליך המלא בממשק — צעד אחר צעד, לכל שלב

### 1. קליטת קריאה חדשה
**אוטומטית (הערוץ הפעיל):** הסנכרון מנתי רץ ומושך פניות פתוחות ← לכל פנייה נוצרות אוטומטית: קריאה (Call), תיק (Case), קישור ללקוח קיים או יצירת לקוח חדש, וקישור לספק אם כבר שובץ בנתי ← הקריאה מופיעה מיד ב"לוח בקרה" וב"רשימת קריאות". הפעלה ידנית של הסנכרון: ניהול נתונים ← "סנכרון נתונים מנתיד" ← אפשר "בדיקה (Dry Run)" לפני.
**ידנית:** לוח בקרה ← כפתור **"+ קריאה חדשה"** ← טופס פתיחה: פרטי מתקשר (טלפון מזהה לקוח קיים אוטומטית), רכב, סוג תקלה, מיקום (עם השלמת ערים), סוג שירות ← שמירה ← הקריאה נכנסת לזרימה הרגילה.

### 2. תור עבודה ושיוך מוקדן
ברגע שקריאה נוצרת (מכל ערוץ): נכנסת אוטומטית ל**תור העבודה** עם ניקוד עדיפות (דחוף/קריטי מקבלים קדימות, לקוח VIP מקבל תוספת) ← המערכת מאתרת את המוקדן **במשמרת** עם העומס הנמוך ביותר ומשייכת אליו ← נשלחת התראה למוקדנים ← במסך "ניטור תורים" / "תצוגת מחלקה" רואים את התור החי, מי מטפל במה, וכמה ממתין. מוקדן בהפסקה — הקריאות הלא-מטופלות שלו מועברות אוטומטית חזרה לתור.

### 3. שיוך ספק
בקריאה ← כפתור **"שבץ ספק"** ← נפתח דיאלוג השיבוץ:
1. בחירת **סוג רכב שירות**: ניידת / גרר / גרר עם אחסנה (רשימת הספקים מסתננת בהתאם).
2. בגרר עם אחסנה: מילוי **מיקום אחסנה + יעד סופי**.
3. בחירת ספק — שלוש דרכים: ידנית מהרשימה, **"שיבוץ אוטומטי"** (מנוע ניקוד לפי מרחק וזמינות בצד השרת), או **המלצת AI**.
4. אישור ← נשלח **SMS אוטומטי לספק** עם פרטי הקריאה ← הסטטוס עובר ל"ספק שובץ".

בקריאה שכבר משובצת, אותו כפתור מציע: **"החלף ספק"** (אותה קריאה) או **"המשך עם ספק נוסף"** (פותח קריאת המשך מקושרת — למשל ניידת שלא צלחה וצריך גרר).

### 4. ניהול הקריאה — מרגע השיבוץ ועד הסגירה (כולל צד לקוח)
במסך הקריאה, **בורר הסטטוס מציג רק את הצעד הבא האפשרי** (ההגדרה של דורית):
1. ספק שובץ ← "**ספק הגיע ללקוח**" או "ביטול פניה".
2. ספק הגיע ללקוח ← "**ספק הגיע לאחסנה**" / "**סגירת קריאה**" / "ביטול פניה".
3. באחסנה ← "סגירת קריאה" (אחסנה נוספת או יעד סופי) / "ביטול פניה".
4. **סגירת קריאה** פותחת דיאלוג עם 7 תוצאות טיפול (ניידת סיימה, גרר הגיע ליעד, גרר לאחסנה...) ← לפי התוצאה: SMS ללקוח, ו/או **קריאת המשך אוטומטית** (למשל אחרי חילוץ), ו/או מעבר לאחסנה.
5. ביטול ← דיאלוג ביטול עם סיבה וכללי עירבון (ביטול במקום = העירבון נשאר).

בכל שלב זמינים בלשוניות: פרטים, כלכלה (עירבון/מוצרים/תמחור), תזכורות, מפה, צ'אט, קבצים/תמונות/חתימה, היסטוריה מלאה.
**צד הלקוח:** SMS אוטומטי בשינויי סטטוס ← מעקב עצמאי ב**פורטל הלקוח** (מזין טלפון + מספר קריאה ורואה סטטוס והתקדמות) ← בסגירה: סקר שביעות רצון ב-SMS.

### 5. מפות וזמינות אונליין
**"מפת ספקים"** בתפריט ← כל נותני השירות על המפה עם סטטוס זמינות (זמין/עסוק/לא מחובר) ← מסייע לשיבוץ לפי קרבה. **בתוך קריאה** ← לשונית "מפה" ← מיקום הספק החי מול נקודת האיסוף, כולל מרחק וזמן הגעה משוער. הספק משתף מיקום מהנייד דרך הפורטל שלו.

### 6. התראות ואיחורים
לוח בקרה ← פאנל **"ניהול איחורים"**: כל קריאה שחצתה את יעד ה-SLA מופיעה עם משך האיחור ותגית "קריטי" ← לחיצה על "צפה" פותחת את הקריאה לטיפול. במקביל: **פעמון התראות** (קריאות ללא שיוך, חריגות SLA), התראות על פקיעת חוזי ספקים ועירבונות (נבדקות אוטומטית), ותשתית התראות Push לנייד.

### 7. דוחות
מסך **"דוחות"** בתפריט ← דוח ביצועי ספקים (זמני הגעה, דירוגים, עמידה ב-SLA), דוח ניצול קריאות, התפלגות סטטוסים וגרפים בלוח הבקרה ← הנתונים נבנים מכל הקריאות כולל 383 ההיסטוריות שיובאו. בנוסף: ייבוא נתונים היסטוריים ממסך ייעודי.

### 8. פורטל ספקים
הספק מתחבר עם המשתמש שלו ← רואה **רק את הקריאות שלו** (תיחום מלא) ← קריאה חדשה מגיעה כ**הצעת שיבוץ** עם פרטי האירוע ← **מאשר או דוחה** ← מרגע האישור מנהל את הקריאה: יצאתי לדרך ← הגעתי ← בטיפול ← סיום, כולל העלאת תמונות והחתמת לקוח ← מנהל גם: פרופיל, זמינות והפסקות, ושיתוף מיקום GPS חי.

## השורה התחתונה

**כל שמונת הרכיבים מההיקף המקורי בנויים וקיימים במערכת.** שלושה מאומתים במלואם בתנאי אמת (קליטה, שיוך ספק, איחורים), ארבעה מאומתים ברובם עם פערים מוגדרים שדורשים יום עבודה אמיתי או מכשיר נייד, ואחד (GPS חי) ממתין לבדיקת שטח. אין רכיב חסר או במצב "לא עובד ידוע".

הפערים המרוכזים ליום/שבוע העבודה הראשון: SMS אמיתי (ספק+לקוח), שיוך מוקדן אוטומטי, GPS מנייד, זרימת ספק מלאה בשטח, ואימות דוחות על נתונים מצטברים.
