## התחלה מהירה

### יוזר חדש - Onboarding מלא

**דרישות מוקדמות:**
- Node.js 18+
- Git
- Claude Code CLI (`npm install -g @anthropic-ai/claude-code`)
- API key של Anthropic (מוגדר כ-`ANTHROPIC_API_KEY`)

**שלבי התקנה:**
```bash
# 1. שכפל את הרפו
git clone <repo-url> && cd natid-crm

# 2. הרץ setup אוטומטי (מתקין תלויות + מכין סביבה)
bash scripts/quick-start.sh

# 3. ערוך את הגדרות הסביבה
#    (הסקריפט יצר .env.local מתוך .env.example - ערוך את הערכים)
nano .env.local

# 4. הפעל את הפרויקט
npm run dev

# 5. פתח Claude Code וקבל הכרות עם הפרויקט
claude
```

**סדר קריאה מומלץ:**

| סדר | מסמך | מה תלמד |
|------|-------|----------|
| 1 | `CLAUDE.md` | הקשר כללי, tech stack, conventions |
| 2 | המסמך הזה (`docs/CLAUDE_WORKFLOW.md`) | תהליך עבודה מלא - 10 פרקטיקות |
| 3 | `SYSTEM_SPECIFICATION.md` | אפיון מלא של המערכת (עברית) |
| 4 | `docs/LESSONS_LEARNED.md` | לקחים שנצברו - מונע טעויות חוזרות |
| 5 | `.claude/skills/` | 8 סקילים מוכנים לשימוש |
| 6 | `docs/WORKFLOWS.md` | תהליכים עסקיים |

**משימה ראשונה מומלצת** - הכרות עם הקודבייס:
```
פתח Claude Code והקלד:
"קרא את CLAUDE.md ואת docs/LESSONS_LEARNED.md.
הסבר לי את מבנה הפרויקט כאילו אני מפתח חדש.
תן לי מפה ויזואלית של הקומפוננטות והמודולים."
```

**משימה שנייה מומלצת** - תיקון באג קטן בתור תרגול:
```
"מצא issue פתוח קל בפרויקט ותכנן את הפתרון.
אל תתחיל לקודד - תן לי קודם את התכנית לאישור."
```

### יוזר קיים - מה השתנה ואיך מתחילים

**מה נוסף:**
- תהליך עבודה מובנה עם Claude Code (10 פרקטיקות)
- **8 סקילים** מוכנים ב-`.claude/skills/` - תבניות לפעולות חוזרות
- סקריפט worktrees - 3 טרמינלים מקביליים בפקודה אחת

- `.claude/settings.json` - הרשאות מוגדרות מראש

**3 דברים להתחיל איתם עכשיו:**

```bash
# 1. הפעל worktrees (עבודה מקבילית - אופציונלי)
bash scripts/worktree-setup.sh setup

# 2. פתח Claude Code
claude

# 3. התחל משימה עם תכנון
"תכנן את [המשימה] לפני שתתחיל לקודד.
רשום קבצים, סדר פעולות, ומה יכול להשתבש."
```

**4 הרגלים ליום-יום:**

| מתי | מה לעשות | הפרומפט |
|------|----------|---------|
| לפני קידוד | תכנן | "תכנן לפני שתקודד" |
| אחרי קידוד | בדוק | "הרץ lint ו-build ותקן שגיאות" |
| אחרי באג | תעד | "עדכן את LESSONS_LEARNED עם הלקח" |

## 1. עבודה מקבילית עם Worktrees

### מה זה?
הרצת 3-5 טרמינלים עם Git Worktrees במקביל. כל worktree = סשן Claude נפרד ובלתי תלוי.

### הגדרה ראשונית
```bash
# הרצה חד-פעמית
bash scripts/worktree-setup.sh setup
```

זה יוצר 3 worktrees:
- `../natid-crm-za` (טרמינל za)
- `../natid-crm-zb` (טרמינל zb)
- `../natid-crm-zc` (טרמינל zc)

### שימוש יומיומי
```bash
# מעבר מהיר בין worktrees
cd ../natid-crm-za   # טרמינל ראשון
cd ../natid-crm-zb   # טרמינל שני
cd ../natid-crm-zc   # טרמינל שלישי
```

### דוגמאות לעבודה מקבילית
| טרמינל | משימה |
|---------|--------|
| za | פיתוח פיצ'ר חדש |
| zb | תיקון באג |
| zc | כתיבת תיעוד / code review |

### כללים חשובים
- כל worktree עובד על branch נפרד
- **לא** לערוך את אותו קובץ בשני worktrees במקביל
- בסיום, merge את הכל ל-main
- נקה worktrees שלא צריך: `bash scripts/worktree-setup.sh cleanup`

## 2. Plan Mode - תכנון לפני יישום

### העיקרון
**משקיעים זמן רב בתכנית ברורה לפני שמקודדים.** Claude מיישם הרבה יותר טוב עם מפרט מדויק.

### תהליך
1. **פתח סשן Claude חדש**
2. **כתוב את התכנית** (השתמש ב-`.claude/plan-template.md`)
3. **הפעל Claude שני לביקורת** ("בדוק את התכנית כמהנדס בכיר")
4. **עדכן את התכנית** לפי ההערות
5. **רק אז** - התחל לקודד

### Template מהיר
```
"אני רוצה [לתאר את המשימה].

לפני שתתחיל לקודד:
1. רשום את כל הקבצים שצריך לשנות
2. תכנן את סדר הפעולות
3. ציין מה יכול להשתבש
4. הצע את הפתרון הכי פשוט

תן לי את התכנית ותחכה לאישור שלי."
```

### ביקורת Senior
```
"בדוק את התכנית הזו כמהנדס בכיר.
מצא חולשות, מקרים קצוניים, ובעיות פוטנציאליות.
אל תהיה עדין - תן feedback כנה."
```

## 4. Skills - פעולות חוזרות

### העיקרון
**כל פעולה שחוזרת על עצמה יותר מפעם-פעמיים הופכת ל-skill קבוע.**

### מיקום
```
.claude/skills/
├── plan-and-review.md     # תכנון וביקורת
├── ci-build-check.md      # בדיקות בנייה
├── update-docs.md         # עדכון מסמכים
├── code-review.md         # ביקורת קוד
├── analytics.md           # ניתוח נתונים
├── learning-mode.md       # מצב למידה
├── subagents.md           # סוכנים משניים
└── prompt-patterns.md     # דפוסי פרומפט
```

### איך ליצור skill חדש
1. זהה פעולה שחוזרת על עצמה
2. צור קובץ `.claude/skills/[name].md`
3. כלול: מתי להשתמש, התהליך, דוגמאות, דגשים
4. הוסף ל-git כדי שכולם יוכלו להשתמש

### שימוש ב-skill
```
"השתמש ב-skill [name] כדי לבצע את המשימה"
```

## סיכום - תהליך עבודה יומי

```
1. פתח 3 טרמינלים עם worktrees (za, zb, zc)
2. בכל טרמינל - הפעל Claude Code
3. התחל עם plan mode - כתוב תכנית
4. בצע ביקורת senior על התכנית
5. יישם לפי התכנית
6. הרץ בדיקות (lint, build)
7. בצע code review
8. עדכן מסמכים עם לקחים
9. commit ו-push
```

**זכור:** מסמכים איכותיים = פחות טעויות. עדכן תמיד.
