# NatID CRM — מדריך הצגה והתנעה ללקוח

> **מטרה:** מסמך אחד שמסביר את המערכת מקצה לקצה — לצוות "נתי שירותי דרך".
> בנוי בשלוש שכבות: (1) מסע לקוח E2E, (2) לפי תפקיד, (3) לפי מודול. בסוף — נספח טכני.
> **קהל:** מעורב — מנהלים/תפעול + אנשי מוצר/פיתוח.

---

## 0. מה זו המערכת — במשפט

מערכת לניהול **קריאות שירות דרך** (גרירה, ניידת שירות, שמשות, רכב חליפי ועוד): מהרגע
שלקוח מתקשר / קריאה נכנסת, דרך שיבוץ נותן שירות (ספק), מעקב בשטח, ועד סגירה ומשוב.

**ארבעה תפקידים:**

| תפקיד | מי | מה עושה |
|-------|-----|---------|
| **מנהל (Admin)** | מנהל מערכת | גישה מלאה: הגדרות, משתמשים, דוחות כספיים, אינטגרציות, סנכרון נתי |
| **מוקדן (Operator)** | מוקד שירות | פותח/מטפל בקריאות, משבץ ספקים, עוקב, סוגר |
| **ספק (Vendor)** | נותן השירות בשטח | מקבל/דוחה שיבוץ, מעדכן סטטוס, מעלה תמונות/חתימה — דרך **פורטל הספקים** |
| **טכנאי (Agent)** | עובד שטח/לקוח | פורטל מצומצם — צפייה בקריאות ופרופיל |

## 1. מסע לקוח — מקצה לקצה (הסיפור המרכזי)

### שלב 1 — קליטת הקריאה
הקריאה יכולה להיכנס מ-**5 מקורות**:
1. **ידני** — מוקדן פותח קריאה במסך "קריאה חדשה".
2. **בוט / IVR** (99digitalBot) — לקוח פותח קריאה דרך בוט.
3. **CTI / מרכזייה** — webhook ממערכת הטלפוניה.
4. **CRM חיצוני** — webhook ממערכת צד ג'.
5. **סנכרון נתי** — סנכרון דו-כיווני אוטומטי (כל ~5 דקות): משיכת קריאות פתוחות של מחלקת הגרירה ממסד הנתונים של נתי, ודחיפת העדכונים שנעשו כאן (סגירות, שיבוץ ספק, זמני הגעה/סיום, הערות מוקד, בקרת איכות) חזרה לנתי.

📍 **סטטוס התחלתי:** `ממתין לטיפול` (waiting_treatment).
🤖 **אוטומציה:** קטגוריזציה אוטומטית של סוג הקריאה, יצירת לקוח אם לא קיים, SMS אישור ללקוח.

### שלב 2 — שיבוץ ספק
המוקדן משבץ נותן שירות. שלוש דרכים:
- **שיבוץ אוטומטי** (autoAssignVendor) — לפי זמינות, מרחק/ETA, התאמת שירות.
- **המלצת AI** (recommendVendor) — המערכת מציעה את הספק המתאים ביותר.
- **שיבוץ ידני** — המוקדן בוחר מרשימת ספקים זמינים.

📍 **סטטוס:** `שיבוץ` (assigning) → נשלחת בקשה לספק.
🤖 **אוטומציה:** חישוב מרחק וזמן הגעה משוער (calculateDistanceAndETA), הודעת שיבוץ לספק (SMS/WhatsApp/Push).

### שלב 3 — תגובת הספק (פורטל הספקים)
הספק מקבל התראה ובפורטל רואה את פרטי הקריאה:
- **קיבל** → הקריאה עוברת ל`ספק בדרך` (vendor_enroute).
- **דחה** → נרשמת סיבת דחייה, והמערכת עוברת לספק הבא / מחזירה למוקדן.

### שלב 4 — עבודת הספק בשטח
הספק מעדכן סטטוס בזמן אמת מהפורטל (גם מובייל):
`נותן השירות בדרך ללקוח` (vendor_enroute) → `נותן השירות הגיע ליעד` (vendor_arrived) → `בטיפול` (in_progress) → `ממתין לשיחת סגירה` (awaiting_closure_call — הקריאה נשארת פתוחה עד שיחת הסגירה עם הלקוח).
🤖 **אוטומציה:** מעקב GPS חי, העלאת תמונות (עם חילוץ נתונים ב-AI), חתימת לקוח, התראות סטטוס ללקוח.

### שלב 5 — סגירת הקריאה
המוקדן/הספק סוגר את הקריאה. כולל (לפי הצורך):
- **סרק (Sarak)** — תיעוד ק"מ ריקים.
- **בקרת איכות (BOY)** — סימון QA.
- **שירות עתידי** — אם השירות נדחה למועד אחר (`שירות עתידי`).
- **תשלום** — תיעוד גבייה (אם נדרש).
📍 **סטטוס סופי:** `סגור` (completed).

### שלב 6 — משוב לקוח
נשלח ללקוח קישור משוב (טוקן ייחודי). הלקוח מדרג ומשאיר הערות → הדירוג מצטבר לדירוג הספק.

```
לקוח → קליטה → שיבוץ ספק → ספק בדרך → הגיע → בטיפול → סגור → משוב
       (מוקדן)   (אוטו/ידני)  (ספק מאשר)  (שטח)          (סגירה)  (לקוח)
```

## 2. לפי תפקיד — מי רואה ומה עושה

### מוקדן (Operator)
- **קריאות** — רשימה, סינון לפי סטטוס/ספק/עדיפות, חיפוש.
- **פרטי קריאה** — מידע, צ'אט, היסטוריה, משוב, קבצים.
- **פעולות** — שיבוץ ספק, שינוי סטטוס, שליחת WhatsApp, תזכורות, ביטול, פתיחה מחדש.
- **תור עבודה / תצוגת מחלקה** — ניהול עומסים.

### ספק (Vendor) — פורטל הספקים
- **קריאות משויכות** — קבלה/דחייה, עדכון סטטוס.
- **זמינות** — מתג זמין/לא זמין/בהפסקה.
- **GPS** — שיתוף מיקום חי.
- **תמונות + חתימה** — תיעוד ביצוע.
- **חוזים / הפסקות / מובייל** — תצוגות ייעודיות.

### מנהל (Admin)
- כל מה שלמוקדן **+** ניהול משתמשים והרשאות, הגדרות, דוחות כספיים, אינטגרציות, סנכרון נתי, ניקוי נתונים.

### טכנאי (Agent)
- פורטל מצומצם — פרופיל, צפייה בקריאות רלוונטיות, התראות.

## 3. לפי מודול / פיצ'ר — טבלת ייחוס

| מודול | מה הוא נותן | טכני (פונקציה/אינטגרציה) |
|-------|------------|--------------------------|
| **שיבוץ אוטומטי** | בחירת ספק לפי זמינות/מרחק | `autoAssignVendor`, `recommendVendor` (AI) |
| **מרחק ו-ETA** | זמן הגעה משוער | `calculateDistanceAndETA`, Google Maps / Leaflet |
| **מעקב GPS** | מיקום ספק חי על מפה | `updateVendorLocation` |
| **התראות** | SMS / WhatsApp / Push | `sendSMS` (Twilio), `sendWhatsApp`, `sendPushNotification` (VAPID) |
| **AI** | סיכום קריאה, ניתוח דפוסים, המלצות | `generateCallSummary`, `analyzeHistoricalPatterns`, `detectSmartAlerts` |
| **משוב** | דירוג לקוח אחרי שירות | `createFeedbackToken`, `sendFeedbackSMS`, `validateAndSubmitFeedback` |
| **דוחות / KPI** | מדדי תפעול וביצועי ספקים | דשבורד, `analyzeVendorPerformance` |
| **צ'אט בקריאה** | תקשורת פנימית על קריאה | ישות `Message` |
| **אינטגרציית נתי** | סנכרון דו-כיווני: משיכת קריאות/ספקים/לקוחות + דחיפת עדכונים חזרה | `syncNatiData` + `pushNatiUpdates`, חיבור MySQL ישיר |
| **פורטל לקוח** | מעקב סטטוס בטלפון+מס' קריאה | `getCustomerPortalData` |

## 4. נספח טכני (לאנשי מוצר/פיתוח)

### ארכיטקטורה
- **Frontend:** React 18 + Vite + Tailwind (RTL מלא, עברית). PWA.
- **State:** React Context + React Query.
- **Backend:** פלטפורמת **Base44** — ~50 פונקציות serverless (TypeScript/Deno).
- **מפות:** Leaflet + OpenStreetMap / Google Maps. **SMS:** Twilio.

### ישויות נתונים מרכזיות
`Call` (קריאה), `Case` (אירוע/תיק), `Customer` (לקוח), `Vendor` (ספק),
`CallAssignmentAttempt` (ניסיון שיבוץ), `CallHistory` (היסטוריה),
`CallFeedback` (משוב), `VendorLocation` (מיקום), `Notification` (התראה).

### מצבי קריאה (state machine)
פתוחים: `ממתין לטיפול`, `ממתין לשיבוץ`, `ספק שובץ`, `נותן השירות בדרך ללקוח`,
`נותן השירות הגיע ליעד`, `בטיפול`, `ממתין לשיחת סגירה`, `שירות עתידי`, `במעקב`,
`באחסנה`, `המשך טיפול`, `המתנה לחיוב`.
השמות והצבעים זהים למסך המוקדנים של נתי (ירוק=ממתין, צהוב=בדרך/בטיפול,
כתום=הגיע, לבן=ממתין לשיחת סגירה, סגול=עתידי).
סגורים: `סגור` (completed), `בוטל` (cancelled).

### אינטגרציות
- **נתי (MySQL):** סנכרון דו-כיווני פעיל — משיכה (מחלקת גרירה) ודחיפת עדכונים חזרה, בכללי בטיחות: סטטוס קדימה-בלבד ומילוי שדות ריקים בלבד (בחירת מוקדנית נתי לעולם לא נדרסת). חיבור דרך IP מאושר (Whitelist) ב-RDS.
- **Twilio:** SMS ללקוח/ספק.
- **WhatsApp:** הודעות שיבוץ ופרטי קריאה.
- **Google Maps:** מרחק, ETA, מפות.
- **Push (VAPID):** התראות דחיפה לפורטל הספקים.

### מקורות אמת לתיעוד נוסף
- `SYSTEM_SPECIFICATION.md` — אפיון מערכת מלא.
- `docs/BUSINESS_WORKFLOWS.md` — תהליכים עסקיים מפורטים.
- `docs/WORKFLOWS.md` — 12 תהליכים ברמת קוד.
- `docs/VENDOR_PORTAL_GUIDE.md` — פורטל הספקים.
- `docs/INTEGRATION_SPEC.md` — אפיון אינטגרציות.
