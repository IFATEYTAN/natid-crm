### מה קורה ברגע שלוחצים "צור קריאה"?

1. **מספר קריאה** נוצר אוטומטית (C-XXXXXXXX)
2. **SLA** מחושב אוטומטית לפי הגדרות הלקוח
3. **סטטוס**: "ממתין לטיפול"
4. המוקדן **מועבר** לדף פרטי הקריאה
5. הקריאה **מופיעה** בדשבורד ובתור העבודה

## 4. שיבוץ ספק אוטומטי

### איך המערכת בוחרת ספק?

```
╔══════════════════════════════════════════════════╗
║              🔍 אלגוריתם שיבוץ ספק              ║
╠══════════════════════════════════════════════════╣
║                                                  ║
║  שלב 1: סינון ראשוני                            ║
║  ✓ ספק פעיל (is_active)                         ║
║  ✓ ספק זמין (availability = 'available')        ║
║  ✓ לא דחה את הקריאה הזו בעבר                    ║
║                                                  ║
║  שלב 2: ניקוד (סה"כ עד ~110 נקודות)            ║
║                                                  ║
║  ┌─────────────────────────────────────────┐     ║
║  │  📍 מרחק (עד 40 נק')                    │     ║
║  │  עד 5 ק"מ = 40 │ עד 10 = 35 │ עד 20 = 25│    ║
║  │  עד 30 = 15    │ עד 50 = 10 │ מעל 50 = 5│    ║
║  └─────────────────────────────────────────┘     ║
║                                                  ║
║  ┌─────────────────────────────────────────┐     ║
║  │  🔧 סוג שירות (עד 20 נק')               │     ║
║  │  ספק מספק את סוג השירות = 20 נק'        │     ║
║  └─────────────────────────────────────────┘     ║
║                                                  ║
║  ┌─────────────────────────────────────────┐     ║
║  │  ⭐ דירוג (עד 20 נק')                   │     ║
║  │  דוגמה: דירוג 4.5/5 = 18 נקודות        │     ║
║  └─────────────────────────────────────────┘     ║
║                                                  ║
║  ┌─────────────────────────────────────────┐     ║
║  │  ⏱️ מהירות תגובה (עד 10 נק')            │     ║
║  │  עד 10 דק' = 10 │ עד 20 = 8 │ עד 30 = 6│    ║
║  └─────────────────────────────────────────┘     ║
║                                                  ║
║  ┌─────────────────────────────────────────┐     ║
║  │  ✅ אחוז השלמה (עד 10 נק')              │     ║
║  │  דוגמה: 95% = 9.5 נקודות               │     ║
║  └─────────────────────────────────────────┘     ║
║                                                  ║
║  ┌─────────────────────────────────────────┐     ║
║  │  📊 בונוסים / קנסות                      │     ║
║  │  תמיכה בסוג הרכב = +5                   │     ║
║  │  פחות מ-3 קריאות היום = +5               │     ║
║  │  פחות מ-5 קריאות היום = +2               │     ║
║  │  יותר מ-10 קריאות היום = -5              │     ║
║  └─────────────────────────────────────────┘     ║
║                                                  ║
║  שלב 3: בחירת הספק עם הניקוד הגבוה ביותר       ║
║  + 3 חלופות מוצגות למוקדן                       ║
║                                                  ║
║  שלב 4: חישוב ETA (OSRM / Google Maps)           ║
║  ראשי: OSRM routing API (ניתוב כבישים אמיתי)    ║
║  חלופי: Google Maps Directions API (cache 4 שע')  ║
║  fallback: (מרחק × 2 דקות) + 5 דקות ביטחון      ║
║                                                  ║
╚══════════════════════════════════════════════════╝
```

### מה קורה אם הספק לא עונה?

```
ספק מספר 1 ←── שיבוץ ──── 2 דקות לאשר
     │                         │
     │    ❌ דחה / ⏰ פג זמן    │
     │                         │
     ▼                         │
ספק מספר 2 ←── שיבוץ מחדש     │   (ספק 1 מוחרג)
     │                         │
     │    ❌ דחה / ⏰ פג זמן    │
     │                         │
     ▼                         │
ספק מספר 3 ←── שיבוץ מחדש     │   (ספקים 1+2 מוחרגים)
     │                         │
     │    ❌ אין ספקים          │
     │                         │
     ▼                         │
📢 הקריאה חוזרת לסטטוס "ממתין לשיבוץ"
   המוקדן צריך לשבץ ידנית
```

## 6. תור העבודה של המוקדן

### איך הקריאות נכנסות לתור?

```
                              קריאה חדשה
                                  │
                                  ▼
                         ┌────────────────┐
                         │ חישוב עדיפות   │
                         │ (50-125 נקודות)│
                         └───────┬────────┘
                                 │
                    ┌────────────┴────────────┐
                    ▼                         ▼
           יש מוקדן פנוי?                אין מוקדן פנוי
          (פחות מ-5 קריאות)             (כולם עמוסים)
                    │                         │
                    ▼                         ▼
           משובץ למוקדן               נכנס לתור המתנה
           עם הכי מעט קריאות          (waiting_in_queue)
```

### מה המוקדן רואה בתור שלו?

**דף "התור שלי"** — מתרענן כל 30 שניות:

| מה מוצג | דוגמה |
|---------|-------|
| סה"כ קריאות | 12 |
| פעילות | 5 |
| הושלמו | 6 |
| דחופות | 1 |

**חלוקה לטאבים:**
- **פעילות** — קריאות שעדיין לא סגורות
- **הושלמו/בוטלו** — קריאות שנסגרו

### מה המנהל רואה ב"ניטור תור"?

**דף "ניטור תור"** — מתרענן כל 15 שניות:

| מדד | הסבר |
|-----|------|
| ממתינים בתור | קריאות ללא מוקדן |
| שובצו למוקדן | יש מוקדן אחראי |
| בטיפול | מוקדן עובד על הקריאה |
| עומס לפי מוקדן | גרף עם כמה קריאות לכל מוקדן |

**קריאה עם ציון עדיפות מעל 80 מסומנת באדום** כדחופה.

## 7. מעקב GPS

### איך זה עובד?

```
  טלפון הספק                    השרת                     המוקדן
  ──────────                    ──────                   ──────
      │                           │                        │
  GPS מופעל                       │                        │
      │                           │                        │
      │── מיקום + מהירות ──▶      │                        │
      │   + דיוק + סוללה         │                        │
      │   (כל 30 שניות)          │                        │
      │                           │                        │
      │                    מעדכן מיקום ספק                  │
      │                    שומר היסטוריה                    │
      │                           │                        │
      │                    אם בקריאה פעילה:                │
      │                    מחשב ETA חדש ───────────▶ רואה על המפה
      │                    מרחק ÷ מהירות                   ETA מעודכן
      │                           │                        │
```

### נתונים שנשלחים

| נתון | הסבר |
|------|------|
| מיקום | קו רוחב + קו אורך |
| דיוק | כמה מטרים סטייה |
| מהירות | קמ"ש |
| כיוון | לאן הספק נוסע |
| סוללה | אחוז טעינה |
| קריאה פעילה | אם הספק בדרך לקריאה |

### חישוב ETA

**שלב שיבוץ ראשוני:** OSRM routing API (ניתוב כבישים אמיתי) או Google Maps Directions API.
Fallback לנוסחה: `(מרחק × 2 דקות) + 5 דקות`.

**עדכון דינמי בזמן נסיעה** (דרך `updateVendorLocation`):
```
ETA = מרחק (ק"מ) ÷ מהירות (קמ"ש) × 60 דקות

אם מהירות < 5 קמ"ש (עומד) → משתמש ב-40 קמ"ש כברירת מחדל
```

כל נתוני ה-GPS עוברים דרך פונקציית שרת `updateVendorLocation` עם אימות בעלות ו-rate limiting (60/דק').
