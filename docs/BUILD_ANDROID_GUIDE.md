# מדריך בנייה: אפליקציית Android (צעד-אחר-צעד)

מדריך זה מסביר איך להפוך את הקוד לאפליקציית Android אמיתית שאפשר להתקין על טלפון —
**בלי ידע בתכנות**. מיועד למי שבונה בעצמו/ה על מחשב Windows או Mac.

> 🍎 **iOS?** גרסת ה-iOS מוכנה בקוד אבל דורשת מחשב Mac. כשיהיה Mac — יש מדריך נפרד
> ב-`docs/LIVE_TRACKING_CAPACITOR_PLAN.md`. המדריך הזה הוא ל-Android בלבד.

---

## מה זה "בנייה" בכלל?
הקוד שכתבנו הוא כמו **מתכון**. "בנייה" (build) זה כמו **לבשל את המנה בפועל** —
להפוך את המתכון לאפליקציה שאפשר להתקין. את הבישול הזה עושים פעם אחת על המחשב שלך,
דרך תוכנה בשם **Android Studio**.

---

## שלב 0 — מה צריך להתקין פעם אחת (30–60 דקות)

### 1. Node.js
- להיכנס ל-<https://nodejs.org> → להוריד את הגרסה הגדולה משמאל (LTS) → להתקין (Next → Next → Finish).

### 2. Android Studio
- להיכנס ל-<https://developer.android.com/studio> → Download → להתקין.
- בפתיחה הראשונה הוא יציע להוריד רכיבים (Android SDK) — **לאשר הכל** וללחוץ Next עד הסוף.
  זה כולל את כל מה שצריך לבנייה (Gradle, SDK, אמולטור).

זהו. את שתי התוכנות האלה מתקינים **פעם אחת בלבד**.

---

## שלב 1 — להוריד את הקוד למחשב

אם כבר יש לך את הקוד על המחשב — דלגי לשלב 2.

אם לא, פותחים חלון פקודות (ב-Windows: "Terminal" או "Command Prompt"; ב-Mac: "Terminal")
ומריצים שורה-שורה:

```bash
git clone https://github.com/IFATEYTAN/natid-crm.git
cd natid-crm
git checkout claude/system-architecture-discussion-wfmpap
```

> השורה האמצעית נכנסת לתיקיית הפרויקט; השלישית בוחרת את הגרסה עם קוד ה-Android.

---

## שלב 2 — פקודה אחת שמכינה הכל

בתוך תיקיית הפרויקט, בחלון הפקודות, מריצים **פעם אחת**:

```bash
npm install
```

(מוריד את כל הרכיבים — לוקח כמה דקות, רץ פעם אחת.)

ואז את הפקודה שבונה את החלק ה-web, מסנכרנת אותו לאפליקציה, ופותחת את Android Studio:

```bash
npm run cap:android
```

Android Studio ייפתח לבד עם הפרויקט. **בפתיחה הראשונה** הוא "מסנכרן Gradle" —
זה סרגל התקדמות למטה שרץ כמה דקות. פשוט מחכים שייגמר. (זה קורה פעם אחת; בפעמים הבאות מהיר.)

## שלב 3 — להתקין על הטלפון

יש שתי אפשרויות. **הכי פשוט להתחיל עם אפשרות א'.**

### אפשרות א' — טלפון אמיתי דרך כבל USB (מומלץ)
כדי לראות שהמעקב ברקע באמת עובד, עדיף טלפון אמיתי:

1. בטלפון Android: הגדרות → "אודות הטלפון" → ללחוץ 7 פעמים על "מספר Build" → זה מפעיל
   "מצב מפתחים".
2. הגדרות → "אפשרויות מפתחים" → להדליק **"ניפוי באגים ב-USB" (USB Debugging)**.
3. לחבר את הטלפון למחשב בכבל. בטלפון תקפוץ הודעה — ללחוץ **"אשר" (Allow)**.
4. ב-Android Studio, למעלה, ליד כפתור ה-▶ הירוק, יופיע שם הטלפון שלך.
5. ללחוץ על כפתור ה-**▶ (Run)** הירוק. האפליקציה תיבנה ותיפתח על הטלפון תוך דקה-שתיים.

### אפשרות ב' — אמולטור (טלפון וירטואלי על המחשב)
אין טלפון פנוי? אפשר טלפון וירטואלי:
1. ב-Android Studio: תפריט **Tools → Device Manager → Create Device** → לבחור דגם → Next → Finish.
2. ללחוץ ▶ ליד האמולטור, ואז את כפתור ה-**▶ (Run)** הירוק.

> הערה: **מעקב מיקום ברקע לא נבדק טוב על אמולטור.** את הבדיקה האמיתית (מסך נעול,
> מעבר ל-Waze) עושים על טלפון פיזי.

## שלב 4 — קובץ התקנה לשיתוף (APK)

רוצה קובץ אחד שאפשר לשלוח בוואטסאפ/מייל ולהתקין על טלפונים אחרים בלי כבל?

1. ב-Android Studio: תפריט **Build → Build App Bundle(s) / APK(s) → Build APK(s)**.
2. כשנגמר, תקפוץ הודעה קטנה בפינה עם קישור **"locate"** — ללחוץ עליו כדי למצוא את הקובץ
   (`app-debug.apk`).
3. לשלוח את הקובץ לטלפון היעד. בטלפון: לפתוח את הקובץ → לאשר "התקנה ממקור לא ידוע" → התקנה.

> זהו קובץ בדיקה (debug). להעלאה **לחנות Google Play** צריך קובץ חתום (signed AAB) +
> חשבון Google Play Developer — זה שלב נפרד שנעשה כשמוכנים להפצה רשמית.

---

## בדיקה שהכל עובד (הסיבה שבנינו את זה)
אחרי שהאפליקציה מותקנת על טלפון אמיתי:
1. להיכנס כספק (טכנאי) ולהדליק את הטוגל **"שתף מיקום"**.
2. לאשר את בקשת ההרשאה — לבחור **"בזמן השימוש"** ואז, כשיישאל, **"אפשר תמיד"**
   (זה מה שמאפשר מעקב ברקע).
3. תופיע התראה קבועה "שיתוף מיקום" — זה תקין ומכוון (Android מחייב אותה).
4. לנעול את המסך / לעבור ל-Waze ולנסוע. במסך המעקב במערכת המיקום צריך להמשיך להתעדכן —
   **גם אחרי 5 דקות** (זה מה שההגדרות `useLegacyBridge` + `CapacitorHttp` פותרות).

## נתקעת?
- **"npm לא מזוהה"** → Node.js לא הותקן / צריך לסגור ולפתוח מחדש את חלון הפקודות.
- **Gradle sync נכשל** → בדרך כלל אינטרנט/פרוקסי. ב-Android Studio: File → Sync Project
  with Gradle Files, ולנסות שוב.
- **הטלפון לא מופיע** → לוודא ש-"ניפוי באגים ב-USB" דלוק ושלחצת "אשר" בטלפון; לנסות כבל אחר.
- כל שגיאה אחרת → לצלם מסך ולשלוח, ואפשר לפתור יחד.
